# 📋 知识推送内容品控 SOP

> 核心原则：**用 AI 的文笔，但锁死 AI 的逻辑**

---

## 🎯 问题根源

AI（大语言模型）最大的弱点是 **"幻觉"**：
- 数学公式看起来像模像样，但代入数值就崩溃
- 游戏案例听起来合理，但工程上根本不可能
- 逻辑推导显得专业，但结论站不住脚

---

## 第一阶段：输入端控制 (Prompt Engineering)

### 核心策略
> **不要让 AI "发明" 知识，而是让它 "转述" 知识。**

### 1. 角色与约束设定

❌ **错误指令：**
```
写一篇关于高等数学极限的趣味文章，用游戏做例子。
```

✅ **正确指令：**
```
你是一位**资深游戏数值策划**兼**数学科普作家**。
请解释'极限'的概念。

**核心约束：**
1. **严谨性：** 所有的数学公式必须是教科书级别的经典案例
   （如 1/x, sin(x)/x, (1+1/n)^n），**严禁自己生造复杂的代数公式**。
2. **通俗性：** 用'王者荣耀的攻速阈值'或'原神的体力恢复'作为类比。
3. **结构：** 采用'痛点引入 -> 核心概念 -> 游戏案例 -> 避坑指南'的结构。
```

### 2. 举例的特殊控制

AI 最容易在举例时翻车。强制它进行**数值验算**：

```
在给出任何公式示例时，请在后台（思维链）中代入 x=0, x=1, x=10, x=∞ 进行验算。
如果结果不符合常理（例如伤害为0、概率超过100%），请更换例子。
```

### 3. 公式来源锁定

**黄金法则**：人工提供真实公式，AI 负责润色排版。

常用的**真实游戏公式**参考：

| 领域 | 公式 | 说明 |
|------|------|------|
| 护甲减伤 | `Reduction = Armor/(Armor+K)` | 极限趋向100%但永远到不了 |
| 复利增长 | `A = P(1+r/n)^(nt)` | 当n→∞时趋向于e |
| 攻速阈值 | `实际攻速 = min(基础×加成, 上限)` | 典型的分段函数 |
| 经验衰减 | `ExpGain = Base × (1/Level^0.5)` | 边际递减 |
| 概率保底 | `P累计 = 1-(1-p)^n` | 极限趋向1 |

## 第二阶段：审核端控制 (Human Loop)

### 核心策略
> **人工只做"逻辑警察"，不做"文字编辑"。**

### 1. 公式代入法 (3秒测试)

看到公式，立刻代入**极端值**：

| 代入值 | 检查项 |
|--------|--------|
| x = 0 | 结果是否有意义？是否除零？ |
| x = 1 | 结果是否符合"默认值"逻辑？ |
| x = 10 | 结果是否在合理范围？ |
| x = ∞ | 结果是否收敛到合理极限？ |

**真实案例（错误）：**
```
D(x) = (100x - 100)/(x - 1)
```
- 代入 x=1：分母为0 ❌ 直接毙掉
- 代入 x=2：结果100。代入 x=100：还是100。❌ 逻辑不通

### 2. 常识冲突法

快速扫描以下红旗：
- ⚠️ 时间是负数？
- ⚠️ 概率大于 1？
- ⚠️ 等级/经验是小数？
- ⚠️ 结论反常识？（如"优化越多帧率越低"）

### 3. 红蓝对抗 (AI 查 AI)

开一个新对话窗口，把生成内容丢进去：

```
我正在审阅这篇文章，请帮我**找茬**。

重点检查：
1. 文中的数学公式是否有逻辑漏洞？
2. 游戏案例是否符合实际开发逻辑？
3. 如果有错误，请直接指出，不要客气。
```

## 第三阶段：呈现端控制 (UX)

### 核心策略
> **适应"手机刷屏"的阅读习惯（3-5分钟注意力）**

### 1. 视觉锚点

| 元素 | 用途 |
|------|------|
| 🎯 💡 ❌ ✅ | 标记重点 |
| **加粗关键词** | 核心结论必须突出 |
| > 引用块 | 金句收藏 |
| 表格 | 对比概念 |

### 2. 碎片化原则

- 每段不超过 3-4 行
- 每个概念配一个类比
- 避免长篇大论

### 3. 交互式结尾

- 留思考题（不直接给答案）
- 设计反直觉的判断题
- 目的：从"被动阅读"转为"主动思考"

## 📝 实战 Checklist

每次生成内容前，走一遍这个清单：

### 准备阶段
- [ ] 确定今日选题
- [ ] 搜索 1-2 个**真实公式/案例**（不让 AI 瞎编）
- [ ] 准备约束型 Prompt

### 生成阶段
- [ ] 使用标准 Prompt 模板
- [ ] 要求 AI 进行数值验算
- [ ] 生成至少 2 个版本对比

### 审核阶段
- [ ] 公式代入极端值测试
- [ ] 常识冲突扫描
- [ ] （可选）AI 红蓝对抗

### 发布阶段
- [ ] 视觉锚点检查
- [ ] 碎片化程度检查
- [ ] 互动题目完整性

## 📚 各学科注意事项

### 🔢 高等数学
- **高危区**：自编公式
- **安全做法**：只用教科书经典案例（sinx/x, e的定义, 等比级数）
- **真实应用**：护甲减伤、复利计算、概率收敛

### 📈 宏观经济学
- **高危区**：瞎编数据、混淆概念
- **安全做法**：引用真实历史事件、官方统计数据
- **核对来源**：世界银行、国家统计局、Fed

### 📊 炒股
- **高危区**：暗示投资建议
- **安全做法**：强调"投资有风险"，只讲概念不荐股
- **法律风险**：避免任何"保证收益"的表述

### 🎨 交互设计
- **高危区**：主观审美当客观标准
- **安全做法**：引用 Nielsen、Fitts's Law 等经典理论
- **案例选择**：优先选大厂产品（可验证）

### 🎮 游戏系统设计
- **高危区**：自编"万能公式"
- **安全做法**：引用具体游戏的已知机制
- **案例选择**：《原神》《王者荣耀》等主流游戏

### 🎵 编曲
- **高危区**：乐理错误
- **安全做法**：引用标准乐理术语
- **验证方式**：实际在DAW中验证和弦/音阶

## 🔧 标准 Prompt 模板

```markdown
**Role:** 资深[领域]专家 & 科普作家
**Goal:** 为非专业人员写一篇关于"[主题]"的早读文章
**Style:** 轻松、幽默、碎片化（类似"罗辑思维"风格）

**Content Requirements:**
1. **Hook:** 用"[具体痛点/有趣现象]"作为引入
2. **Concept:** 用最直白的话解释核心概念，不堆砌术语
3. **Case Study (关键):** 使用以下真实案例：
   [在此粘贴你准备好的真实公式/案例]
   **严禁自行编造公式或数据**
4. **Validation:** 对所有数学公式，代入 x=0,1,10,∞ 验算
5. **Format:**
   - 使用 Emoji 列表
   - 每段不超过 3-4 行
   - 核心结论加粗
   - 结尾给出思考题

**Output Structure:**
1. 📖 故事引入（痛点 Hook）
2. 🎯 核心概念（3句话说清楚）
3. 🔍 深入分析（真实案例）
4. 💡 重点记忆（表格/口诀）
5. ✨ 金句收藏（可发朋友圈）
6. 📝 课后练习（分离题目/答案）
```
